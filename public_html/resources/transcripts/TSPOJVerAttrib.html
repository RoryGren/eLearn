Project and Version Attributes

An Attribute is "additional" information for a particular record in PowerOffice that can help with

  • filtering,
  • categorizing and placing of information in reports, and
  • prioritizing of applications in the toolbar.

Project Version Attributes inherit the first three values from the parent Project Attributes. I.e.

  • Project Type,
  • Project Engineering Stage and
  • Project Approval Stage.

These values can be changed and do not necessarily need to be the same as those of the parent.

Changing the values is done by highlighting the Attribute in the grid and making any changes at the bottom of the dialog box, either by clicking the drop-down arrow and making a selection form the list provided, or, if there is no list, typing the required value into the field, and then finally clicking the Update Value button.

The fourth attribute value is particularly important for projects using the PO Electrification application. The value here determines the folder name to be used for project-specific files and rules that PowerOffice will create and use.

Material Reservation Status is only to be used by Eskom. It must remain Not Ready until the project is complete and a reservation needs to be made in SAP.
Once this is changed to Ready, the selected Project Version will be locked for editing and no more changes will be possible.

The remaining attributes pertain to costing and require an annual percentage to be entered to be added to the material cost of the selected version.
