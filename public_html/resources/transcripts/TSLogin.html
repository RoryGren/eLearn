Logging in to PowerOffice.

Logging in to PowerOffice is a simple process, but can be frustrating if a user forgets his or her password!

  • Fill in the correct Login Name and Password. These are user defined and will probably be allocated to you by your organisation.
    • Make sure you don't forget your password. If you do, an administrator will need to re-set your password for you. (There is a default User Name with no password. This should only be used for the initial setup. Thereafter, only use the User Name allocated to you by your organisation.)
  • Make sure the correct server installation is selected by putting the radio button dot in the correct position. (Most users will use Microsoft SQL Server - MSSQL)
  • This will change the server block below to request the relevant parameters for connecting to the database.
    • For MSSQL Server:
      • Server Name requires the name of the computer where the database engine has been installed.
        • If you have a stand-alone installation, i.e. everything will run on your computer and there will be no other computer connecting to PowerOffice on your computer across a network, you will type in the name of your own computer here - the local computer. The exact name.
          PowerOffice will suggest the name of the local computer for you.
        • If you have a client-server setup, i.e. you have a dedicated server which runs the PowerOffice MSSQL database engine and your computer connects to it across a network, you will type in the name, or the IP address of the server here. The exact name. If necessary, your IT support team can help you here.
      • Instance requires just that, the MSSQL Instance on the server. With PowerOffice, this is IIPower (Not "Two Power", the letter 'I' - Aye-aye Power!)
        If you have a dedicated server, your administrator will need to open up a 'Port' in the server firewall to allow outside computers to access the database engine.The users must then "tell" their computer what port to use on the server.
        This is done by following the Instance with a comma and the port number - no spaces.
      • Database requires the database name to connect to, inside the instance of MSSQL. The PowerOffice 14 database will be PowerOffice14.
    • For Oracle:
      • ORA DB(SID) requires the database SID. This is like the database name on the server.
      • Server Name this requires the name of the computer where the database engine has been installed.
        • If you have a stand-alone installation, i.e. everything will run on your computer and there will be no other computer connecting to PowerOffice on your computer across a network, you will type in the name of your own computer here - the local computer. The exact name.
          PowerOffice will suggest the name of the local computer for you.
        • If you have a client-server setup, i.e. you have a dedicated server which runs the PowerOffice Oracle database engine and your computer connects to it across a network, you will type in the name, or the IP address of the server here. The exact name. If necessary, your IT support team can help you here.
      • Port requires the port number of the port that is opened for outside computers to access your database. PowerOffice defaults to port number 1521.
