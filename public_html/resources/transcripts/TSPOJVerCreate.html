Creating a Project Version

Versions are created inside Projects. Projects are containers for sub-projects and Versions.

The version will be created inside the project that is selected or highlighted. (Remember - PowerOffice works around what is selected.)

On the Toolbar, click the black drop-down arrow to the right of the Project button and select Version.

This opens the Version dialog box.

In the Version dialog box, we have the following:

Auto Code Generation
  • this option is sometimes not visible, depending on user rights assigned to the user.
  • With this option checked, the Code and Description fields will be greyed out / inactive and PowerOffice will generate a Version Code which is unique within the parent project, usually something like "V1" and a Description which will be something like "Version 1."
  • With this option unchecked, the Code and Description fields will be active and PowerOffice will allow the user to fill in the relevant information. (These fields must NOT be left blank.)

The Code is the Unique identifier of the new version within the project.

The Description is exactly that, the description. It is what humans will find more understandable to identify the version.

The Cost Source and Cost Version are linked to the Item Cost module and enable the user to select or use groups or sets of costs to cost the version against. (These will be discussed in more detail in chapter 4.)

Clicking Create, creates the Version and opens the Project Version Attributes dialog box. (These will be discussed in the next section - Version Attributes.)

Clicking OK on closes the Version Attributes dialog and allows the user to modify the fields in the Version dialog box, if necessary.

Clicking OK here will create the new version and close the Version dialog box.

Right Clicking a version and selecting Edit will open the Version dialog box and allow the user to change / Edit the fields.
